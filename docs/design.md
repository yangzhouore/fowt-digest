# Design Direction

## Visual Position

The site should feel restrained, editorial, and academic. Use https://ai-digest.liziran.com/en/ only as inspiration for clarity, typography, spacing, numbered content, and linear reading. Do not copy its branding, content, layout details, or exact styling.

## Reading Experience

Prioritize long-form readability over visual novelty. Pages should guide readers through a clear sequence:

1. page context;
2. digest or paper metadata;
3. concise analysis;
4. limitations and source links.

Use numbered sections where they help scanning, especially on weekly edition pages.

## Typography & Layout

Use semantic HTML, clear typographic hierarchy, generous whitespace, and readable line lengths. The layout should work well on mobile and desktop without requiring complex interaction. Prefer simple page sections with thin borders and direct headings.

Recommended visual traits:

- high contrast body text;
- restrained colour palette;
- visible source and metadata labels;
- accessible links;
- consistent spacing rhythm;
- responsive single-column reading on small screens.

## Avoided Treatments

Do not use:

- rounded content cards;
- gradients;
- glassmorphism;
- decorative shadows;
- decorative animations;
- carousels;
- dashboard-style layouts;
- excessive icons;
- UI component libraries.

## Components

Create components only when repeated structure justifies them. Likely future components include a paper summary row, metadata list, edition navigation, and source link block. Do not turn every page section into a component.

## Content Integrity

The design should make source attribution and mock status obvious. Readers must be able to distinguish journal papers, conference papers, and preprints without relying on colour alone.
